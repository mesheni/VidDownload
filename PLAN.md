# План развития VidDownload

## Категория A — Критические баги

| # | Проблема | Файл:строка |
|---|----------|-------------|
| **A1** | **`.gitignore` содержит markdown-ограждение** — файл начинается с `\`\`\`gitignore` и заканчивается `\`\`\``, из-за чего git не может корректно распарсить правила игнорирования. | `.gitignore:1,45` |
| **A2** | **`WebClient.DownloadFileAsync` не ожидается** — `CheckUpdateAsync()` запускает асинхронную загрузку yt-dlp и сразу выходит. Будет исправлено в D2 (очередь загрузок): при отсутствии ссылки на видео загрузка просто не начнётся. | `MainWindow.xaml.cs:295-298` |

---

## Категория B — Серьёзные баги и утечки

| # | Проблема | Файл:строка |
|---|----------|-------------|
| **B1** | **Утечка ресурсов** — `FileStream` и `StreamWriter` создаются без `using`, закрываются только в happy-path. При исключении логи не flushed, файл остаётся открытым. | `MainWindow.xaml.cs:113,141` |
| **B2** | **Деление на ноль в прогрессе FFmpeg** — `TotalLength` может быть `TimeSpan.Zero` для «живых» потоков. | `FFmpegAction.cs:92` |
| **B3** | **Нет проверки существования yt-dlp.exe перед запуском** — если обновление не скачалось, `Process.Start` кинет необработанное исключение. | `MainWindow.xaml.cs:118-120` |
| **B4** | **Блокирующий `.Result` на UI-потоке** — `CheckForInternetConnection().Result` в `CheckUpdateAsync` может привести к дедлоку в некоторых контекстах. | `MainWindow.xaml.cs:237` |

## Категория D — Новый функционал

| # | Фича | Приоритет |
|---|------|-----------|
| **D1** | **Кнопка отмены загрузки** — `CancellationToken` + кнопка «Стоп». | Высокий |
| **D2** | **Очередь загрузок** — добавление нескольких ссылок и последовательная обработка. В отдельном окне/пространстве очереди отображать название и иконку видео. При отсутствии ссылок — ничего не происходит, что также решает A2. | Высокий |
| **D3** | **Сохранение пользовательских настроек** — последние выбранные разрешение/кодек/формат между запусками (через `Properties.Settings` или JSON). | Средний |
| **D4** | **Тёмная/светлая тема** — переключение тем через HandyControl. | Низкий |
| **D5** | **Локализация на английский** — ресурсные файлы `.resx`, переключение языка. | Низкий |

## Рекомендуемый порядок реализации

1. **A1** — исправление `.gitignore` (тривиально)
2. **C5** — миграция на .NET 10 (фундамент)
3. **D2** — очередь загрузок (ключевая фича, закрывает A2)
4. **C1, C2, C3** — рефакторинг: `HttpClient`, `UpdateService`, `DownloadService`
5. **D1** — кнопка отмены загрузки (на базе C3 + D2)
6. **B1-B4** — исправление багов параллельно с рефакторингом
7. **D3** — сохранение настроек
8. **C4, C6, C7, C8** — чистка кода
9. **D4, D5** — темы и локализация по потребности
